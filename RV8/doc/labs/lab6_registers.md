# Lab 6: Register File

## Objective
Build the CPU registers (a0, t0, sp, pg) and connect a0 to the ALU feedback loop.

## Components
| Part | Qty | Description |
|------|:---:|-------------|
| 74HC574 | 4 | Octal D flip-flop (a0, t0, sp, pg) |
| LED + 330Ω | 8 | a0 output display |

## Concept

```
         ┌──────────────────────────────────┐
         │                                  │
         ▼                                  │
    ┌─────────┐     ┌─────┐     ┌──────┐   │
    │ a0 (U7) │────►│ ALU │────►│result│───┘
    └─────────┘     └─────┘     └──────┘
         ▲               ▲
         │               │
    a0_clk          B input (from IR operand or register)
```

The accumulator (a0) feeds the ALU, and the ALU result writes back to a0.
This creates the compute loop: a0 = a0 OP B.

## Schematic

```
U7 (a0):
  D[7:0] ← ALU result [7:0]
  CLK ← a0_clk (manual pushbutton for now)
  /OE ← GND
  Q[7:0] → ALU input A + LEDs

U8 (t0):
  D[7:0] ← data bus
  CLK ← t0_clk (manual pushbutton)
  /OE ← GND
  Q[7:0] → (available as ALU input B via mux, later)

U9 (sp):
  D[7:0] ← data bus
  CLK ← sp_clk (manual pushbutton)
  /OE ← GND
  Q[7:0] → stack address (used in Step 7)

U10 (pg):
  D[7:0] ← data bus
  CLK ← pg_clk (manual pushbutton)
  /OE ← GND
  Q[7:0] → page register (used in Step 7)
```

**What to check in GTKWave:**

---

## Procedure

1. Insert U7–U10 (74HC574). Connect VCC (pin 20) and GND (pin 10).
2. U7 (a0): wire D inputs from ALU result outputs.
3. U7 (a0): wire Q outputs to ALU input A (replace DIP switches from Lab 5).
4. U7 (a0): wire Q outputs to 8 LEDs.
5. U7 CLK: connect to a pushbutton (with pull-down, rising-edge triggers latch).
6. Tie U7 /OE to GND (always outputting).
7. Wire U8, U9, U10 similarly (D from data bus, CLK from individual buttons).
8. Remove DIP switches from ALU input A (now driven by a0).

## Test Procedure

| Test | Action | Expected Result |
|:----:|--------|-----------------|
| 1 | Set operand=$05 (on U6 or switches), SUB=0 | ALU shows a0+$05 |
| 2 | Press a0_clk | a0 latches result. LEDs show $05 (0+5) |
| 3 | Press a0_clk again | a0 = $0A (5+5) |
| 4 | Press a0_clk again | a0 = $0F (10+5) |
| 5 | Change operand to $01 | ALU shows a0+1 |
| 6 | Press a0_clk repeatedly | a0 counts: $10, $11, $12... |
| 7 | Set SUB=1, operand=$01 | ALU shows a0-1 |
| 8 | Press a0_clk | a0 decrements |
| 9 | RESET (clear a0) | a0 = $00, LEDs off |

## Checkoff

- [ ] a0 latches ALU result on clock edge
- [ ] Feedback loop works: a0 accumulates (adds repeatedly)
- [ ] SUB mode: a0 decrements correctly
- [ ] Overflow wraps: $FF + $01 = $00
- [ ] RESET clears a0 to $00

## Notes
- This is the core compute loop of the CPU. Every ALU instruction does: a0 = a0 OP operand.
- In the final build, a0_clk is generated by the control unit (not a manual button).
- The other registers (t0, sp, pg) are loaded from the data bus, not from the ALU.
- t0 is used as a second operand source: `ADD t0` means a0 = a0 + t0.

## เป้าหมาย

สร้าง register a0 (accumulator) และต่อเข้ากับ ALU เป็นวงรอบ — ให้ CPU คำนวณซ้ำได้

## ความรู้พื้นฐาน

**Accumulator (a0)** = ตัวเก็บผลลัพธ์หลัก ทุกการคำนวณจะเป็น: a0 = a0 OP ค่าอะไรสักอย่าง

**วงรอบคำนวณ:**
```
a0 ──► ALU ──► ผลลัพธ์ ──► กลับเข้า a0
```
ทุกครั้งที่กด clock ผลลัพธ์ใหม่จะถูกเก็บใน a0

**ตัวอย่าง:** ถ้า operand = 5 และกด clock 3 ครั้ง:
- ครั้งที่ 1: a0 = 0 + 5 = 5
- ครั้งที่ 2: a0 = 5 + 5 = 10
- ครั้งที่ 3: a0 = 10 + 5 = 15

## ขั้นตอนต่อวงจร

1. เสียบ U7–U10 (74HC574) ต่อ VCC (pin 20) และ GND (pin 10)
2. U7 (a0): ต่อ D จาก ALU result
3. U7 (a0): ต่อ Q ไปที่ ALU input A (แทน DIP switch จาก Lab 5)
4. U7 (a0): ต่อ Q ไปที่ LED 8 ดวง

7. ถอด DIP switch ออกจาก ALU input A (ตอนนี้ a0 ขับแทน)
